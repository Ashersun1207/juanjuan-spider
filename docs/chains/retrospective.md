# 复盘链 — 阶段性复盘

---

## v0.1-v0.5 复盘（2026-02-26）

### 做对了
- "不造轮子"原则从头贯彻：Crawl4AI + trafilatura 都是成熟项目
- 双引擎路由：静态页不启动浏览器，性能差 10 倍
- 三层去噪比单层效果好得多
- SQLite+文件双写兼顾查询和可读性
- MCP 复用 main.crawl() 而非独立实现（虽然一开始犯了 L9 的错）

### 做错了
- v0.1 自写引擎花了时间，早该直接用 Crawl4AI
- MCP Server 最初独立实现管道，trafilatura 集成后才发现 MCP 没同步（L9）
- adapter 覆盖 fit_markdown 的 bug 到 v0.5 才发现（L7）
- CSS 选择器适配器太激进导致 0 字符输出（L5）

### 下次改进
- 新项目第一天就选成熟框架，不先自写再换
- 所有入口共享核心函数，从第一行代码开始
- adapter 做最小干预（只 transform raw），不碰引擎产出的字段
